## HabitTrack Mini

Мини-приложение для отслеживания привычек и ежедневных активностей. Пользователь добавляет привычки, отмечает выполнение, смотрит историю и получает мотивирующую цитату из публичного API.

### Основные функции (MVP)
- Добавление новой привычки (название, целевая частота, опционально цвет/иконка).
- Список привычек с индикаторами выполнения за сегодня.
- Отметка выполнения привычки за день, отмена отметки.
- Экран «Детали привычки»: краткая статистика по последним дням.
- Получение мотивационной цитаты из публичного API (для ЛР6, пример: `https://api.adviceslip.com/advice`).
- Базовая стилизация: светлая тема, акцентный цвет для привычек.

### Структура экранов
- **Главный экран**: список привычек, кнопка «+», быстрые переключатели дней.
- **Экран добавления/редактирования**: поля ввода, выбор цвета/иконки, сохранение.
- **Детали привычки**: прогресс за последние 7 дней, кнопка «Получить цитату», отображение ответа API.

### Навигация
- Главный → Добавление/редактирование (кнопка «+» или тап по карточке).
- Главный → Детали привычки (тап по имени/строке привычки).
- Детали → Назад на главный (кнопка Back/стрелка).

### Эскизы (схематично)
- Главный: `[+ Add]` | Список карточек: `✅ Чтение (сегодня выполнено)` / `○ Спорт (не выполнено)`.
- Добавление: `Название [____]`, `Частота [ежедневно/другие]`, `Цвет/иконка`, `Сохранить`.
- Детали: `Привычка: Спорт`, `График/чипы дней: ✅✅○✅○○✅`, `Кнопка: Получить цитату`, `Блок ответа API`.

### Сценарии использования
- **Создание привычки**: Пользователь нажимает «+», вводит данные, сохраняет → привычка появляется в списке.
- **Отметка выполнения**: На главном экране тап по чекбоксу/иконке статуса → состояние на сегодня переключается.
- **Просмотр деталей**: Тап по привычке → экран деталей → виден прогресс, кнопка «Получить цитату».
- **Получение цитаты**: На деталях нажать кнопку → запрос к `api.adviceslip.com` → показать текст совета/ошибку.

### План по лабораторным работам
- **ЛР4 (верстка)**: Разметка всех экранов, навигация между ними, базовая тема/цвета, моковые данные в состоянии экрана.
- **ЛР5 (логика)**: Управление состоянием списка привычек, добавление/редактирование, переключение статуса выполнения, валидация ввода.
- **ЛР6 (данные)**: Хранение привычек и отметок (сначала in-memory, затем `shared_preferences`/локальная БД), вызов публичного API за цитатой, обработка ошибок сети/лоадеров, кэш последней цитаты.

### Чек-лист (соответствие требованиям)
- Главный + дополнительный экран (детали/добавление) — есть.
- Навигация между экранами — описана.
- Ввод пользователя — формы добавления/редактирования.
- Список данных — список привычек.
- Хранение данных — in-memory → persistent (ЛР6).
- Работа с API — запрос цитаты (`api.adviceslip.com` или аналогичный публичный).
- Стилизация — базовая светлая тема + акцентные цвета для привычек.
